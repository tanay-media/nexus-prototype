IDE MCP (OAuth)
1OAuth only (same SSO as this app)

MCP connects with OAuth 2.1 (PKCE) through the same identity provider as the web app. API-key auth is not supported for MCP — do not add X-API-Key headers in your client config.

Stay signed in here in the browser. The first MCP tool call from Cursor or Claude opens a browser window to finish authorization if your IDE doesn't already have a valid token.

2Choose environment

Production is the default. Point at staging only for testing.

Production · https://mcp.nexus.app/sse
Staging · https://mcp.staging.nexus.app/sse
3Cursor (remote URL — no npx)

Add to ~/.cursor/mcp.json:

{
  "mcpServers": {
    "nexus": {
      "url": "https://mcp.nexus.app/sse"
    }
  }
}
4Claude Code (HTTP transport — not the same as Cursor)

Run in your project root:

claude mcp add --transport http nexus https://mcp.nexus.app/mcp
5Claude Desktop (npx + mcp-remote bridge)

Add to claude_desktop_config.json:

{
  "mcpServers": {
    "nexus": {
      "command": "npx",
      "args": ["mcp-remote", "https://mcp.nexus.app/sse"]
    }
  }
}
